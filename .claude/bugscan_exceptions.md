# Bugscan Exceptions

Known issues, accepted risks, and false positives that should NOT be reported in future scans.
Each entry should be a concise one-liner describing what to skip and why.

## Accepted Risks
- pd/np NameError when AutoGluon not installed (deployment edge case — AutoGluon is always installed in production)
- Default admin password Admin123! with no forced rotation (warning printed to stdout, acceptable for internal tool)
- Session tokens stored plaintext in sessions.json on disk (filesystem access = game over anyway)
- Server binds 0.0.0.0 by default (configurable via HOST env var, standard for containerized deployments)
- No user deletion/deactivation capability for active users (feature gap, not a bug)
- No CSRF token on POST endpoints (mitigated by SameSite=Strict cookies)
- SameSite cookie not supported in older browsers (irrelevant for internal tool with modern browsers)
- time.time() used for rate limiting instead of time.monotonic() (NTP jumps negligible for minute/hour windows)
- SimpleRateLimiter / APIRateLimiter memory growth (bounded by total unique users, small for internal tool)
- No pagination on get_all_models (bounded by 50-model-per-user quota)
- Spin-wait in model_cache.load_model with time.sleep(0.2) (suboptimal but functional, low real-world impact)
- _session_store._clean_expired O(n) on every session create (bounded by session count, internal tool)
- _safe_read_json creates unlimited .corrupt.bak files (requires persistent corruption, extremely unlikely)
- Orphan model directory cleanup could hit in-progress training (1-hour guard is generous)
- In-memory Airflow ZIP export can use significant RAM for large models (rare admin operation)
- No max length on audio transcripts stored in model metadata (acceptable at call center scale)
- Global 200MB upload limit may block multi-file audio uploads (generous for typical call center audio)
- handle_change_password verifies password against potentially stale data (race requires same user from two sessions simultaneously)
- Airflow ZIP export size check only measures agmodel directory, not full ZIP (minor, model files dominate size)
- _embedding_cache_evictor daemon thread started at module import time (server.py never imported as library)
- _session_store.periodic_flush thread started at module import time (same — never imported as library)
- ResourceManager instantiated at module scope with GPU/RAM probing (same — never imported as library)
- Blob URL minor memory leaks in frontend download functions (cleaned up on page navigation)

## False Positives
- _cors_allowed as class variable evaluated once at import (standard pattern for env-var config, not a bug)
- escAttr double-encoding & character (Step 1 JS doesn't touch &, Step 2 HTML encodes it — correct order)
- _safe_filename allows Unicode in Content-Disposition (intentional, prevents response splitting only)
- Multipart parser strips trailing -- from file content (only triggers for malformed bodies; well-formed bodies have closing marker in separate section)
- Whisper double-checked locking race returns None (min_idle_seconds guard makes extremely unlikely, error caught per-file)
- LLM prompt injection via schema variable names (users control their own evaluation schemas, no cross-user impact)
- Wilson CI math.sqrt negative argument (p clamped to [0,1], expression always >= 0)
- feature_columns_embedded not saved to metadata (conditional save matches conditional use — both gated on text_columns)
- handle_predict semaphore release without matching acquire (return before try block, finally never executes)
- torch NameError in _ensure_vram_available second try block (caught by except Exception)
- clean_dataframe context="preview" undefined behavior (correct fallthrough to standard tabular cleaning)
- parseCSVLine doesn't handle CRLF in quotes (quote-aware line splitter handles this upstream)
- renderMd regex injection via pre-escaped content (esc() called first, all captures are safe)
- Frontend doesn't handle 503 on upload semaphore (api() receives JSON body and displays error correctly)
- handle_predict_batch double body Content-Type read (single header read per handler, not a bug)
- _per_model_locks unbounded growth (non-existent models get throwaway RLock, not cached)
- _atomic_write_json temp file naming collision (all writes serialized by _file_locks RLock)
- find_user returns mutable reference to dict (load_users re-reads from disk, fresh list each call)
- _ensure_vram_available returns stale VRAM check (defensive code, subsequent operations verify)
- Audio temp dir not cleaned up on error (cleanup IS in finally block, handles all paths)
- CORS wildcard + credentials ambiguity (code correctly omits Allow-Credentials when wildcard)
- JobStore.__getitem__ non-atomic mutations (GIL makes dict __setitem__ atomic in CPython)
- Integer overflow in _wilson_ci for large datasets (math is sound with clamped inputs)
- Unquoted HTML class attribute in renderCallAnalysisResults (valid HTML, no spaces in class name)
- Feature input element ID collision on similar column names (values collected via data-feature attribute, not getElementById)
